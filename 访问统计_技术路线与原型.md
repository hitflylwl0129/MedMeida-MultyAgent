**前端 track.js 自动识别**，不需要每个 HTML 手动配置。
板块字典写在 `track.js` 里集中维护：

```js
const SECTION_DICT = {
  // 同站不同端口
  '162.14.76.209': { default: 'marketing' },                // 80 端口
  '162.14.76.209:8000': {                                   // ocr-agent 8000 端口
    '/ocr':  'ocr',
    '/asr':  'asr',
    '/raw':  'raw',
    '/qc':   'qc',
  },
};

const SUBSECTION_BY_PATH = {  // 营销中台下钻
  '/app.html':        'home',
  '/':                'home',
  '/product.html':    'product',
  '/doctor.html':     'doctor',
  '/script.html':     'script',
  '/video.html':      'video',
  '/distribute.html': 'distribute',
  '/audience.html':   'audience',
  '/admin.html':      'admin',
};

function detectSection(){
  const host = location.host;
  const path = location.pathname;
  const dict = SECTION_DICT[host];
  if (!dict) return 'unknown';
  if (dict.default){
    // 营销中台主域 → 板块=marketing，子板块从 SUBSECTION_BY_PATH 取
    return {
      section:    dict.default,
      subsection: SUBSECTION_BY_PATH[path] || 'unknown',
    };
  }
  // 8000 端口的 ocr/asr/raw/qc：path 前缀匹配
  for (const [prefix, id] of Object.entries(dict)){
    if (path.startsWith(prefix)) return { section: id, subsection: '' };
  }
  return { section: 'unknown', subsection: '' };
}
```

### 2.3 跨端口埋点的关键问题：CORS

`http://162.14.76.209:8000/ocr/` 页面要向 `http://162.14.76.209/api/track/visit` 上报 →
**端口不同 = 不同源**，浏览器拦截。

**两个解法**：

| 方案 | 优点 | 缺点 | 推荐度 |
|---|---|---|---|
| 后端 FastAPI 加 CORS 中间件，allow_origin=`['http://162.14.76.209:8000']` | 标准方式 | 需写 CORS 头；ocr/asr 用户会看到 OPTIONS 预检请求 | ★★★ |
| 用 `<img src="...">` 1px GIF 跟踪（**无跨域限制**）；上报通过 query string | 浏览器原生支持，无 CORS / 无 OPTIONS / 无 fetch | 只能 GET，data 走 URL（最大 ~2KB 够用） | ★★★★ |

**推荐 GIF 跟踪法**（PR-2 实现）：

```js
function report(payload){
  const qs = new URLSearchParams(payload).toString();
  // 1x1 透明 GIF，浏览器不会渲染，但服务端能收到完整请求
  new Image().src = 'http://162.14.76.209/api/track/p.gif?' + qs;
}
```

服务端 `/api/track/p.gif` 返回 `Content-Type: image/gif` 的 1 字节透明 GIF，写完数据库就 return。
**leave 事件**仍用 `navigator.sendBeacon()`（也无 CORS 限制 + 不阻塞页面卸载）。

这套组合下 **PR-2 完全不需要 CORS 配置**，更省事。

## 3. 三套技术路线对比

### 路线 A：纯 nginx access_log + 离线分析（最轻）

- ✅ **零侵入**，nginx 已经在记 access_log
- ✅ 完全无前端改动
- ❌ **拿不到停留时长**（只有请求时间戳）
- ❌ **拿不到 SPA 内的板块切换**（如果未来 product.html 内有标签页）
- ❌ admin 后台只能看"请求"，不是"用户行为"
- 适用：只想看"哪些 IP 来过、各板块访问量"，不要停留时长

### 路线 B：前端 SDK 埋点 + 后端 API 入库（推荐）★

```
浏览器                    后端 FastAPI                  存储
─────────                 ─────────────                ─────────
1. <script src="/static/track.js">
   ├─ pageshow → POST /api/track/visit                 sqlite: events 表
   │  body = {page, referrer, ts, session_id}          字段：ip/ua/page/event/ts/dur
   ├─ 心跳 30s → POST /api/track/heartbeat             
   ├─ pagehide → POST /api/track/leave  (用 sendBeacon  
   │            不阻塞页面跳转)                        
   └─ 路由切换 → POST /api/track/visit (SPA hash 变化时)
                    ↑
                    nginx 转发时已经把 X-Real-IP 注入
```

- ✅ 拿得到完整"进入 → 停留 → 离开"链路
- ✅ 板块切换可埋点（哪怕 SPA 也能追）
- ✅ 后端可以加 GeoIP / 浏览器指纹 / 频次限流
- ✅ 跟现有 FastAPI / SQLite 完全同栈
- ⚠️ 需要给每个 HTML 注入一行 `<script>`（统一注入，9 个文件）
- ⚠️ 略增加流量（每用户 ~1KB/分钟）

### 路线 C：第三方分析（百度统计 / GA / Plausible）

- ✅ **0 工作量**，加一行 script 即可
- ✅ 仪表盘、热图、漏斗等高级功能现成
- ❌ **数据出域**（敏感 demo 站不合适）
- ❌ **后台 UI 不是你的**，无法定制"板块"维度
- ❌ 你说要"做后台页面"，已暗示要自己控制 UI → 不选 C

### 推荐：**路线 B**

理由：你 demo 是医药营销 + 选品业务 + 沙箱 + 数智人，**数据敏感**不能出域；同时你**已经在用 FastAPI + SQLite + 同源前端**，B 路线工程一致性最高。

## 4. 数据模型（SQLite，单表设计）

```sql
CREATE TABLE access_events (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    -- 来源
    ip              TEXT NOT NULL,            -- 来自 X-Real-IP
    ip_city         TEXT,                     -- GeoIP 解析（可选）
    ip_isp          TEXT,                     -- 运营商（可选）
    ua              TEXT,                     -- User-Agent（截断 200 字）
    ua_browser      TEXT,                     -- 解析后的浏览器名（Chrome 121）
    ua_os           TEXT,                     -- 操作系统（macOS / Windows）
    ua_device       TEXT,                     -- desktop / mobile / tablet
    -- 会话
    session_id      TEXT NOT NULL,            -- 浏览器侧生成（首次访问写 sessionStorage）
    referrer        TEXT,                     -- document.referrer
    -- 板块
    section         TEXT NOT NULL,            -- 板块 ID（见第 2 节字典）
    page            TEXT NOT NULL,            -- 完整 URL pathname
    title           TEXT,                     -- document.title
    -- 事件
    event           TEXT NOT NULL,            -- 'visit' | 'heartbeat' | 'leave'
    -- 时间
    ts              INTEGER NOT NULL,         -- Unix 秒（事件触发时间）
    dur_sec         INTEGER DEFAULT 0,        -- 停留秒数（leave 事件填，visit 时为 0）
    -- 原始
    raw             TEXT                      -- JSON 序列化的原始上报体（排查用）
);

CREATE INDEX idx_events_ts      ON access_events(ts);
CREATE INDEX idx_events_ip      ON access_events(ip);
CREATE INDEX idx_events_session ON access_events(session_id);
CREATE INDEX idx_events_section ON access_events(section);
```

**估算容量**：单 demo 站点日均 < 10k 事件 / ~5MB SQLite，90 天 ~450MB。SQLite 完全够用。

**为什么不用现有 jobs.db**：访问统计是高频写、可独立删表；与业务数据隔离更安全。文件单独：`backend/.cache/access.db`。

## 5. 关键算法说明

### 5.1 IP 获取（nginx 已配好）
```nginx
proxy_set_header X-Real-IP $remote_addr;          # ✅ 当前 nginx 已有
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
```
后端：`request.headers.get("X-Real-IP") or request.client.host`，**禁用代理头伪造**（白名单只允许 127.0.0.1 上游）。

### 5.2 停留时长（最容易踩坑）
```text
visit ──30s── heartbeat ──30s── heartbeat ──── pagehide(leave)
  │                                              │
  ts_visit                                       ts_leave
                                                 ↓
                                     dur_sec = ts_leave - ts_visit
```

- **正常关闭**：浏览器 `pagehide` 事件 → `navigator.sendBeacon('/api/track/leave', {dur})` 不阻塞导航
- **异常关闭**（断网/崩溃）：服务端定时任务每 5 分钟扫"最后心跳 > 90 秒前的会话" → 用最后心跳时间结算
- **心跳间隔**：30s（流量与精度平衡）；超过 5 分钟不心跳视为离开

### 5.3 板块归一
前端 `track.js` 一个函数：

```js
function detectSection(){
  const path = location.pathname.toLowerCase();
  if (path === '/' || path.endsWith('/app.html')) return 'home';
  if (path.endsWith('/product.html'))      return 'product';
  if (path.endsWith('/doctor.html'))       return 'doctor';
  if (path.endsWith('/script.html'))       return 'script';
  if (path.endsWith('/video.html'))        return 'video';
  if (path.endsWith('/distribute.html'))   return 'distribute';
  if (path.endsWith('/audience.html'))     return 'audience';
  // 跨域名子站：靠 location.hostname 判断
  if (location.hostname.includes('ruyifanyi.com')){
    if (path.includes('/ocr')) return 'ocr';
    if (path.includes('/asr')) return 'asr';
    if (path.includes('/raw')) return 'raw';
    if (path.includes('/qc'))  return 'qc';
  }
  return 'unknown';
}
```

后端有备份判断（API 调用时也能识别板块）。

### 5.4 唯一访客（UV）算法
- **session_id**：浏览器首次进入 → `crypto.randomUUID()` 写 `sessionStorage`，关闭浏览器即失效
- **UV = distinct(ip + ua_hash)**，按天滚动；不放 cookie，**符合 GDPR 的"严格必要"豁免**
- 可选升级：localStorage 保存"访客 ID"（更精准但需要 cookie 同意书）

## 6. 隐私与安全

| 维度 | 措施 |
|---|---|
| **数据出域** | 全部本地 SQLite，不走第三方 |
| **PII** | 不存身份证/手机号；UA 只截 200 字；referrer 截 500 字 |
| **IP** | 存完整 IPv4，admin 页可选脱敏（`192.168.1.***`）一键开关 |
| **后台鉴权** | BasicAuth 或 URL token，**绝不**前端可见 |
| **CSRF/防刷** | `/api/track/*` 只接受 POST + 同源 Referer + 5 req/sec/IP 限流 |
| **隐私声明** | 页脚 strip 旁加 `i 图标` → 弹窗说明"本站统计访问 IP / 浏览类型，不存 cookie，不识别个人" |
| **GeoIP 库** | MaxMind GeoLite2-City 免费版本地化使用 |

## 13. 部署矩阵（用户拍板后的实操路径）

5 个板块的埋点 / 上报路径完整图：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      浏览器（用户访问任意板块）                            │
└─────────────────────────────────────────────────────────────────────────┘
                                  │
       ┌──────────────────────────┼───────────────────────────────┐
       ▼                          ▼                               ▼
 http://162.14.76.209/      http://162.14.76.209:8000/      http://162.14.76.209/
   app.html                   ocr/  asr/  raw/  qc/           admin.html
   product.html                                               （后台 BasicAuth）
   doctor.html                  ↑
   ... 9 个 HTML                这 4 个由 ocr-web.service (:8000)
                                返回 HTML 串
                                ↓
   ↑                            nginx sub_filter 注入：
   prototype 目录里             '</body>' →

       │ 所有页面 │ track.js │ 都上报到同一处 │
       ▼                                            ▼
   ┌─────────────────────────────────────────────────────┐
   │  http://162.14.76.209/api/track/p.gif?...           │
   │  （nginx 80 → 反代到 video-agent.service:8001）     │
   │  ↑ 用 1px GIF 上报，0 CORS 问题                     │
   │                                                     │
   │  FastAPI access_router.py                           │
```

### 各板块的部署改动清单（极简）

| 板块 | 改动文件 | 改动内容 | 责任主体 |
|---|---|---|---|
| `marketing.*`（9 个 HTML） | `prototype/app.html` 等 | 每个 `</body>` 前加 1 行 `<script src="/static/track.js"></script>` | video-agent 仓库 |
| `ocr` / `asr` / `raw` / `qc` | `/etc/nginx/sites-enabled/ocr-agent`（猜测的 nginx 配置） | `location ~* ^/(ocr|asr|raw|qc)/ { sub_filter '</body>' '...track.js...</body>'; sub_filter_once on; }` | nginx 配置 |
| 后端接收 | `backend/app/access_router.py` 新增 | `/api/track/p.gif` 接收上报、写 SQLite | video-agent 仓库 |
| 后台页 | `prototype/admin.html` | 静态原型 → PR-3 接真实 API | video-agent 仓库 |
| 静态资源 | `prototype/static/track.js` 新增 | 前端埋点 SDK | video-agent 仓库 |

**关键洞察**：用户的"5 个板块"全部在同一台服务器，所以：
- ❌ 不需要 ruyifanyi.com 域名相关的跨域配置（之前我以为要做的）
- ❌ 不需要 ocr-agent 后端改动（nginx sub_filter 注入）
- ❌ 不需要 CORS 中间件（用 1px GIF + sendBeacon 全绕开）
- ✅ 只动 video-agent 仓库（新增 access_router / track.js / 注入 9 个 HTML）+ 改一处 nginx 配置加 sub_filter
