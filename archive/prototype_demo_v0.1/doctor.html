⚠ 核心认知：选医生不是"挑粉丝最多的"。本 Agent 把产品功效 + 目标人群约束编码为检索向量，从认证医生库做召回 → 向量匹配 → LLM 精排，在内容垂类 / 人群重合 / 合规历史 / 商业转化多目标间做加权平衡，并支持按策略实时重排。

合规 & 商业约束

  • 硬门槛：医师执业资质
  • 硬门槛：近半年无违规

  • 排除：利益冲突 / 竞品代言
资质前置校验
retrieval_query = {
  "text": "消化内科/营养科医生, 肠道菌群·益生菌科普, 粉丝以25-45岁女性为主, 专业亲和科普风格",
  "hard_filter": { 医师资质:required, 近半年违规:0, 竞品代言:exclude },
  "embedding": [0.18, -0.42, 0.77, … ] (768d 语义向量)
}
🧠 LLM 融合结论：把产品功效域、目标人群画像、合规商业约束融合为一条带硬过滤条件的语义检索 Query，并生成 768 维向量用于后续 ANN 召回——这是"选医生"区别于人工筛号的关键一步。

3多因子向量打分（每位候选 · 5 维量化评分 0-100）
📐 每个维度都有可解释来源：垂类匹配=内容向量余弦相似度；人群重合=粉丝画像与目标人群 Jaccard；合规历史=违规/资质评分；商业转化=历史带货 ROI；风格契合=口播风格匹配。下一步按策略加权得到精排总分。
