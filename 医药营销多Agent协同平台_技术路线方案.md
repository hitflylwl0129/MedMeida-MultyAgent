# 医药/泛健康营销 多Agent协同平台 — 技术路线方案

> 版本：v0.1（技术路线调研稿）
> 日期：2026-06-01
> 范围：根据产品→选医生→选人群→生成话术→生成短视频→分发，全链路多 Agent 自动化流水线

---

## 0. 一句话结论

技术上完全可行，推荐采用 **"编排层（LangGraph 状态机）+ 5 个解耦 Agent + 统一数据/资产中台 + 人审关卡（Human-in-the-loop）"** 的架构。
**但本场景最大的不是技术问题，而是合规问题**——"医生 + 药品/保健品 + 短视频宣讲 + 平台投放"踩在监管最敏感的红线上。技术方案必须围绕合规来设计，否则会一键封号甚至处罚。请先读第 1 节。

---

## 1. ⚠️ 合规红线（必须先对齐，决定产品形态）

这是本项目的**第一约束条件**，直接影响每个 Agent 能做什么、不能做什么。

| 法规/动作 | 关键限制 | 对方案的影响 |
|---|---|---|
| 《互联网广告管理办法》（2023.5 施行） | **禁止**以介绍健康、养生知识等形式变相发布"三品一械"（药品/医疗器械/保健食品/特医食品）广告 | 医生不能"科普包装带货"，话术 Agent 必须内置违禁词/变相广告检测 |
| 《广告法》第16/18条 | 药品/保健品广告**不得**利用广告代言人（含医生、患者）作推荐证明；保健品须标"非药物替代" | "医生口播带货"模式高风险，需改造为合规形态（见下） |
| 国家卫健委（2023起 3 年严查） | **严查医务人员直播带货 / 变相代言** | 真人签约医生直接出镜推药 → 高概率违规 |
| 《医疗广告认定指南》（2025.8） | 明确医疗广告认定边界 | 内容审核 Agent 需对齐此指南做分类 |
| 三品一械广告修订（2026.5 征求意见中） | 监管进一步收紧 | 需预留合规规则可热更新能力 |

**结论与建议形态（三选一，按风险递增）：**

- **A 方案（推荐，低风险）**：定位为「合规营销内容生产 + 投放中台」，药品/保健品按"三品一械"广告正规报备流程走（需广告审查批准文号），医生仅做**知识科普类内容**且与具体产品强隔离；带货链路走品牌蓝V/达人合作的合规通道。
- **B 方案（中风险）**：仅服务**普通食品/化妆品/器械中的一类**等监管较松品类，回避"药品+医生代言"组合。
- **C 方案（高风险，不建议）**：真人医生直接出镜口播推荐具体药品/保健品——基本等于违规，不要做。

> 技术方案下文按"可支撑 A/B 方案"来设计，把**合规审查作为一等公民 Agent**贯穿全流程。

## 2. 整体架构

### 2.1 分层视图

```
┌─────────────────────────────────────────────────────────────┐
│  接入层  Web 控制台 / 开放 API / 任务调度                      │
├─────────────────────────────────────────────────────────────┤
│  编排层  Orchestrator（LangGraph 状态机 + 任务队列）           │
│          管线定义 / 状态流转 / 重试 / 人审关卡 / 可观测         │
├─────────────────────────────────────────────────────────────┤
│  Agent 层（解耦、可独立部署、统一 Agent 协议）                  │
│   ① 选医生  ② 选人群  ③ 话术  ④ 短视频制作  ⑤ 合规审查(横切)   │
├─────────────────────────────────────────────────────────────┤
│  能力层  LLM 网关 / 数字人&视频生成 / 检索RAG / 规则引擎        │
├─────────────────────────────────────────────────────────────┤
│  数据中台  产品库 / 医生库 / 人群标签 / 话术库 / 素材资产库      │
├─────────────────────────────────────────────────────────────┤
│  对接层  巨量引擎/磁力/微信 投放&人群API / 内容平台发布API       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心设计原则

1. **解耦**：每个 Agent 是独立服务，对外暴露标准接口（输入契约/输出契约），可单独迭代、替换、灰度。
2. **流水线 + 状态机**：用 LangGraph 把 5 个 Agent 串成有向图，支持串行/并行/条件分支/回环（如审核不过自动打回话术 Agent）。
3. **Human-in-the-loop**：在"选医生确认""话术终审""视频发布前"设置人工卡点，合规场景必须保留人审。
4. **数据驱动**：所有 Agent 共享统一数据中台，决策可追溯、可复用。
5. **合规横切**：合规审查 Agent 不是流水线末端，而是在 ②③④ 每个产出节点都做拦截。

## 3. 流水线编排（Pipeline）

```
[产品输入]
   │
   ▼
①选医生Agent ──(候选医生Top-N)──► 人审确认
   │
   ▼
②目标人群Agent ──(人群画像+标签包)──► 合规检查
   │
   ▼
③话术Agent ──(分人群话术草稿)──► ⑤合规审查 ──不过──► 回退③
   │ 过
   ▼
④短视频Agent (脚本→分镜→生成→合成) ──► ⑤合规审查 ──► 人工终审
   │ 过
   ▼
[分发Agent] 多平台发布 + 人群定向投放 + 回流数据采集
   │
   ▼
[效果数据] ──► 反馈优化（医生/人群/话术 打分迭代）
```

编排要点：
- 节点间传递结构化 `TaskContext`（产品ID、医生ID、人群包ID、话术版本、素材版本、审核状态）。
- 支持**断点续跑**、**单节点重跑**、**版本回溯**。
- 失败/审核不过自动路由回上游节点，形成闭环。

## 4. 各 Agent 详细设计

### ① 选医生 Agent（产品 → 匹配医生）

**目标**：根据要推广的产品，从签约医生库匹配最合适的 Top-N 医生。

**输入**：产品信息（品类、适用人群、功效宣称、合规边界、目标平台）
**输出**：候选医生列表 + 匹配理由 + 匹配分

**技术实现（推荐"召回+精排"两段式，而非纯 LLM）：**
1. **结构化召回**：基于规则/向量检索，从医生库按 `科室/专长领域/内容垂类/粉丝画像/历史合规记录/出镜意愿` 召回。
2. **语义匹配**：把产品的适用场景与医生擅长领域做 Embedding 相似度匹配（产品标签 ↔ 医生标签）。
3. **LLM 精排+解释**：LLM 综合多维度打分并产出"为什么匹配"的可解释理由。
4. **合规预筛**：过滤掉资质不符、有违规记录、品类不匹配的医生。

**关键数据（医生库字段）**：基础信息、执业资质、擅长领域标签、内容风格、历史投放效果、粉丝画像、合规等级、出镜/授权意愿。

> 合规提示：医生与产品的关联必须满足"科普隔离"，避免形成"医生为某药代言"的事实。

---

### ② 目标人群筛选 Agent（产品+医生 → 目标人群）

**目标**：生成目标人群画像，并落地为各平台可投放的"人群包"。

**输入**：产品、确定的医生、营销目标
**输出**：人群画像（年龄/性别/地域/兴趣/行为/消费层级）+ 各平台人群定向包

**技术实现：**
1. **画像生成**：LLM + 产品知识 + 医生粉丝画像，推导目标人群特征。
2. **标签映射**：把内部人群特征**映射**到各平台官方标签体系（巨量/磁力/微信广告各有标签字典，需维护映射表）。
3. **平台对接**：
   - 抖音/今日头条/西瓜 → **巨量引擎开放平台**（DMP 人群管理 API、莱卡定向、自定义人群包上传）
   - 快手 → **磁力引擎** 开放能力
   - 微信视频号/朋友圈 → **微信广告（腾讯广告）** API
4. **第三方用户系统打通**：通过 OAuth/API 对接；自有一方数据可通过 DMP 上传加密 ID（OAID/IMEI-MD5/手机号 MD5）做人群包，再用平台标签扩展（Lookalike）。

**关键约束**：
- 各平台**不会**把原始用户标签数据导出给你，只能"上传种子人群 + 用平台定向能力放大"。所以方案是"**生成定向策略 + 调投放 API**"，而非"拿到平台用户明细"。
- 医疗/三品一械投放**资质门槛**：开广告户需行业资质、产品审查批文，部分品类限制定向。

---

### ③ 推荐话术 Agent（医生+人群 → 营销话术）

**目标**：为"指定医生 × 指定人群"生成口播/文案话术。

**输入**：产品卖点、医生风格、目标人群画像、平台调性
**输出**：分人群的话术脚本（含合规版本标注）

**技术实现：**
1. **RAG 检索**：检索产品合规说明、批准的宣称口径、医生历史高赞话术、平台爆款结构。
2. **风格对齐**：用医生历史内容做 Few-shot，使话术贴合医生人设语气。
3. **人群适配**：同一产品对不同人群生成差异化话术（痛点/语气/钩子不同）。
4. **合规改写**：内置违禁词库 + 变相广告检测，自动规避"治疗/治愈/最/国家级"等违规表述，标注风险点。

> 这是合规风险最集中的 Agent，必须强制经过 ⑤ 合规审查，且话术库与"批准宣称口径"绑定。

---

### ④ 短视频制作 Agent（脚本 → 分镜 → 生成 → 合成）

**目标**：把话术变成可发布的短视频。

**子流程（可拆成子 Agent / 工具链）：**
1. **脚本结构化**：话术 → 分镜脚本（镜头、时长、台词、字幕、BGM、口播节奏）。
2. **分镜/Storyboard**：文生图生成分镜参考图；或匹配素材库。
3. **视频生成**（按真实度/成本分级）：
   - **数字人口播**（最契合本场景）：阿里云虚拟数字人 / 通义灵眸 / 通义万相 Wan2.2-S2V（图+音生成口播视频）/ 商汤如影 / 腾讯智影 等。
   - **真人医生授权形象**：基于医生授权的形象/声音克隆生成口播（须本人书面授权 + 合规）。
   - **文生视频/图生视频**：可灵、即梦、通义万相等做空镜/演示片段。
   - **TTS 配音**：语音合成 + 口型同步（lip-sync）。
4. **合成与后期**：字幕、贴纸、片头片尾、品牌包装、自动剪辑（FFmpeg/云剪辑）。
5. **多版本产出**：按平台规格（竖屏/时长/封面）批量出片，支持 A/B 多版本。

**技术选型建议**：优先用**云厂商数字人 API（阿里云/腾讯智影）**做工程化集成，自研可控部分放在脚本/分镜/合成编排上。

---

### ⑤ 合规审查 Agent（横切，贯穿②③④）

**目标**：在每个产出节点拦截违规内容。

**技术实现：**
- **规则引擎**：违禁词库（《广告法》绝对化用语、三品一械违禁宣称）+ 正则/词典 + 可热更新规则。
- **LLM 审核**：对话术/脚本/成片字幕做语义级违规判定（变相代言、暗示疗效、虚假宣称）。
- **多模态审核**：成片做画面/语音转写后再审（OCR 字幕 + ASR 口播）。
- **分级处置**：通过 / 警告（标注风险）/ 打回（自动路由回上游）/ 阻断（禁止发布）。
- **可审计**：留存每次审核的依据与版本，便于追溯与监管举证。

---

### 分发 Agent（发布 + 投放 + 回流）

- **内容发布**：抖音开放平台、快手、微信视频号的内容发布 API（视频上传、定时发布）。
- **广告投放**：巨量引擎/磁力/腾讯广告的计划创建、人群定向、预算出价 API。
- **数据回流**：播放/互动/转化数据回采，进入效果分析，反哺①②③的打分模型。

## 5. 技术选型一览

| 模块 | 推荐选型 | 备选 / 说明 |
|---|---|---|
| Agent 编排 | **LangGraph**（状态机、人审关卡、回环、可观测最强） | CrewAI（角色协作、上手快）/ AutoGen（多智能体对话）；复杂工业流水线选 LangGraph |
| LLM | 国产合规大模型（通义/混元/DeepSeek/豆包）经统一网关 | 医疗合规场景建议私有化/专有部署，数据不出域 |
| 检索/RAG | 向量库（Milvus/PgVector）+ Rerank | 医生库、产品库、话术库统一接入 |
| 数字人/视频 | 阿里云虚拟数字人 / 腾讯智影 / 通义万相 Wan2.2-S2V | 可灵、即梦做空镜；自研做编排与合成 |
| TTS/口型 | 云厂商 TTS + lip-sync | — |
| 人群/投放 | 巨量引擎开放平台 / 磁力引擎 / 腾讯广告 API | 受资质与平台政策约束 |
| 视频后期 | FFmpeg / 云剪辑服务 | 模板化批量出片 |
| 任务/消息 | 任务队列（Celery/Temporal）+ 对象存储 | Temporal 适合长流程、可重放 |
| 可观测 | LangSmith / 自建 Trace + 日志 | Agent 全链路追踪 |

## 6. 数据中台（贯穿全局）

- **产品库**：品类、卖点、合规宣称口径、广告审查批文、禁用词。
- **医生库**：资质、专长标签、风格、粉丝画像、授权状态、合规等级、历史效果。
- **人群标签库**：内部标签 ↔ 各平台标签映射表、人群包 ID。
- **话术库**：版本化、按医生/人群/平台分类、合规标注。
- **素材资产库**：分镜图、数字人形象、配音、成片、多版本管理。
- **效果库**：投放与互动数据，驱动反馈优化。

## 7. 分阶段落地路径

| 阶段 | 目标 | 交付物 | 周期建议 |
|---|---|---|---|
| **P0 合规与可行性** | 明确产品形态（A/B/C）、品类、资质、平台政策 | 合规方案 + 法务确认 | 1-2 周 |
| **P1 单 Agent MVP** | 跑通"③话术 + ⑤合规审查"最小闭环 | 话术生成+审核 Demo | 2-3 周 |
| **P2 内容生产线** | 接入"④短视频/数字人"，话术→成片 | 自动出片 Demo | 3-4 周 |
| **P3 编排打通** | LangGraph 串起①②③④⑤ + 人审关卡 | 端到端流水线 | 3-4 周 |
| **P4 平台对接** | 人群包/发布/投放 API 接入（受资质进度影响） | 分发闭环 + 数据回流 | 4-6 周 |
| **P5 反馈优化** | 效果数据反哺医生/人群/话术打分 | 自优化闭环 | 持续 |

> 建议先做 P1+P2 验证"AI 生产合规内容"的核心价值，平台投放（P4）依赖资质审批，并行推进。

## 8. 主要风险与对策

| 风险 | 等级 | 对策 |
|---|---|---|
| 医生代言/变相广告违规 | 🔴 高 | 法务前置；科普与产品强隔离；合规 Agent 强拦截；保留人审 |
| 平台不开放用户明细标签 | 🟠 中 | 改为"种子人群上传 + 平台定向放大"，而非获取明细 |
| 三品一械投放资质门槛 | 🟠 中 | 提前办理广告资质与审查批文，按品类规划 |
| 数字人/克隆形象授权与肖像权 | 🟠 中 | 医生书面授权；声音/形象克隆需明确授权范围 |
| 内容真实性/疗效虚假宣称 | 🔴 高 | 宣称口径绑定批准文案；多模态审核 |
| 生成内容质量与平台限流 | 🟡 低 | 模板+人审+A/B；贴合平台规则 |

---

## 9. 给决策者的建议

1. **先定形态再开发**：P0 合规结论决定整个产品边界，务必拉法务/合规一起定。
2. **技术上分两条腿**：A) "AI 合规内容生产线"（确定性高、可快速验证）；B) "平台投放对接"（依赖资质、政策，周期长）——并行推进、互不阻塞。
3. **合规即护城河**：在监管收紧（2026.5 新规征求意见）的趋势下，谁能把"合规审查"做成产品能力，谁就有壁垒。
4. **不要赌 C 方案**（真人医生直推药品），违规成本远超收益。
