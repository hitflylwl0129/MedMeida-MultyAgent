## 3. 云端 nginx 改造（关键，让 ocr/asr/raw/qc 自动注入 track.js）

### 3.1 改 ocr-web.service：监听换内部端口

```bash
# 备份
cp /etc/systemd/system/ocr-web.service /etc/systemd/system/ocr-web.service.bak.20260612_102351

# 把 0.0.0.0:8000 改成 127.0.0.1:8002
sed -i.bak \
  's|--host 0.0.0.0 --port 8000|--host 127.0.0.1 --port 8002|' \
  /etc/systemd/system/ocr-web.service

systemctl daemon-reload
systemctl restart ocr-web.service
```

改动一行（验证：`grep ExecStart /etc/systemd/system/ocr-web.service`）。

### 3.2 新建 nginx server 块：监听 :8000，反代 :8002，sub_filter 注入

文件：`/etc/nginx/conf.d/ocr-agent-track.conf`

```nginx
# 2026-06-12: 把原 ocr-agent 0.0.0.0:8000 改成 127.0.0.1:8002，
# nginx 在 8000 反代 + sub_filter 注入访问统计 track.js
# 用户原访问形式 http://162.14.76.209:8000/{ocr,asr,raw,qc}/ 完全不变
server {
    listen 8000 default_server;
    server_name _;

    client_max_body_size 200m;

    sub_filter_once on;
    sub_filter '</body>' '<script src="//162.14.76.209/static/track.js" defer></script>\n</body>';

    location / {
        proxy_pass http://127.0.0.1:8002;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 关闭压缩，保证 sub_filter 能匹配明文 </body>
        proxy_set_header Accept-Encoding "";

        # SSE / 长连接
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
        chunked_transfer_encoding on;
    }
}
```

```bash
nginx -t                # 校验
systemctl reload nginx  # reload
```

### 3.3 验证

```bash
# 直连 8002（ocr-agent 内部）：应 200
curl -sS -o /dev/null -w 'HTTP %{http_code}\n' http://127.0.0.1:8002/ocr/

# 经 nginx 8000：应 200 + 含 track.js
curl -sS http://127.0.0.1:8000/ocr/ | grep -c 'static/track.js'   # 应输出 1
```

## 4. 一键回滚（紧急时）

如果访问统计 v1.0 出问题影响 ocr-agent 主业务：

```bash
ssh root@162.14.76.209

# 恢复 ocr-web.service（监听回 0.0.0.0:8000）
cp /etc/systemd/system/ocr-web.service.bak.20260612_102351 \
   /etc/systemd/system/ocr-web.service
systemctl daemon-reload
systemctl restart ocr-web.service

# 删掉新增的 nginx server 块
rm /etc/nginx/conf.d/ocr-agent-track.conf
nginx -t && systemctl reload nginx

# （可选）video-agent 也想关掉访问统计：
# .env 加：ACCESS_STATS_ENABLED=false
# 重启：systemctl restart video-agent.service
```

回滚完成后：
- ocr-agent 重新监听 `0.0.0.0:8000` 直接对外，原行为恢复
- video-agent backend 还在跑（404 端点 /api/track/* 不会再被前端调用，因为 track.js 也消失了）
