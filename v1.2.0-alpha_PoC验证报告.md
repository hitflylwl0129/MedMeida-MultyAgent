## 2. 踩过的两个坑（已固化解决）

### 坑 1：e2b SDK 2.x 客户端强制校验 API Key 格式
- **现象**：直接装 `e2b==2.26.0`，调 `Sandbox.create()` 报
  `AuthenticationException: Invalid API key format: expected "e2b_" followed by hex characters`
- **根因**：e2b 2.x 在 `e2b/api/__init__.py` 写死了正则 `\Ae2b_[0-9a-f]+\Z`；
  而腾讯云 AGR 发的 Key 是 `ark_xxx` 前缀
- **解决**：在 `sandbox_executor._ensure_env()` 启动时 monkey-patch 这个函数为 noop，
  把校验下放给真正的服务端（AGR）做。已加幂等保护 + 日志
- **官方文档没说**：腾讯云 quickstart 用最新 SDK 风格写，但没钉 PyPI 版本，
  导致按官方文档直接装最新版必踩坑

### 坑 2：e2b 1.x 降级方案不可行
- **现象**：降到 `e2b==1.11.1`，发现没有 `Sandbox.create()` 类方法
- **结论**：1.x 是 `Sandbox(template=...)` 构造，腾讯云 quickstart 示例代码风格其实是 2.x 的
- **决策**：用 2.x + monkey-patch（上面方案），不回退 1.x

## 3. 工程落点（不动 v1.1 视频/话术/分发链路）

| 文件 | 用途 |

| `backend/requirements.txt` | 增 `e2b-code-interpreter>=1.0` / `e2b>=1.0` / `playwright>=1.45` |
| `backend/scripts/agr_smoke.py`（新增） | 端到端烟测脚本，留作回归 |

### 关键设计：业务层只看 `sandbox_executor`
```python
from app.sandbox_executor import code_sandbox, browser_sandbox

with code_sandbox() as sb:
    sb.run_code("import pandas as pd; ...")

with browser_sandbox() as (sb, cdp_url, novnc_url):
    # Playwright connect_over_cdp(cdp_url, ...)
    ...
```
